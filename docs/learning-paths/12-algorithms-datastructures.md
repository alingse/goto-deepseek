## 学习方法建议

### 1. 五遍刷题法
```
第1遍：看题，思考5-10分钟
第2遍：看题解，理解思路
第3遍：自己写代码，通过测试
第4遍：24小时后，重写代码
第5遍：一周后，复习总结
```

### 2. 分类刷题
- 按数据结构分类（数组、链表、树、图）
- 按算法分类（DP、贪心、回溯、DFS/BFS）
- 按难度分类（Easy→Medium→Hard）
- 按公司分类（字节、腾讯、阿里等）

### 3. 总结模板
```cpp
// 二分查找模板
int binarySearch(vector<int>& nums, int target) {
    int left = 0, right = nums.size() - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] == target) return mid;
        if (nums[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return -1;
}

// DFS模板
void dfs(TreeNode* root) {
    if (!root) return;
    // 前序位置
    dfs(root->left);
    // 中序位置
    dfs(root->right);
    // 后序位置
}

// 回溯模板
void backtrack(vector<int>& path, vector<vector<int>>& res) {
    if (满足条件) {
        res.push_back(path);
        return;
    }
    for (选择 : 选择列表) {
        做选择;
        backtrack(path, res);
        撤销选择;
    }
}
```

### 4. 错题本管理
- 记录不会做的题目
- 记录做错的题目
- 记录优化空间大的题目
- 定期复习错题

### 5. 模拟面试
- 每天进行1-2次模拟面试
- 严格限时（Medium题15分钟）
- 口述解题思路
- 写出Bug-free代码

## 常见问题与解决方案

### Q1：刷题没有思路怎么办？
**A**：解决步骤：
1. 先看标签，知道考什么
2. 读题5分钟，思考暴力解法
3. 看题解，理解思路
4. 关闭题解，自己重写
5. 总结题型和解法模板

### Q2：如何提高刷题速度？
**A**：提速技巧：
1. 总结常见模板（二分、DFS、DP）
2. 记忆常见题型（双指针、滑动窗口）
3. 练习快速识别考点
4. 提高打字速度
5. 使用快捷键

### Q3：Hard题怎么突破？
**A**：突破方法：
1. 先理解题意，画图分析
2. 从简单情况入手
3. 参考题解思路
4. 分步实现，逐步优化
5. 总结题型规律

### Q4：刷多少题才能面试？
**A**：建议数量：
- **基础**：100题（LeetCode Hot 100）
- **进阶**：200题（按分类刷题）
- **精通**：300-500题（覆盖所有题型）
- **质量 > 数量**：每题都要吃透

### Q5：DP题目怎么想？
**A**：DP解题套路：
1. 定义DP数组含义
2. 找出状态转移方程
3. 确定初始条件和边界
4. 确定遍历顺序
5. 举例推导DP数组

### Q6：图题目怎么做？
**A**：图题框架：
1. 建图（邻接矩阵/邻接表）
2. 选择遍历方式（DFS/BFS）
3. 处理特殊情况（环、孤立点）
4. 应用图算法（最短路、最小生成树）

### Q7：如何准备面试？
**A**：面试准备：
1. 刷LeetCode Hot 100
2. 刷目标公司高频题
3. 手写常见算法
4. 模拟面试练习
5. 整理项目经验
